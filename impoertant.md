● 好问题。它们不是同一个模型，是两个独立的小网络：

  ┌───────────────┬──────────────┬──────────────────┬────────────────────────────────┐
  │     模型      │   训练脚本   │       作用       │              输出              │
  ├───────────────┼──────────────┼──────────────────┼────────────────────────────────┤
  │ BiasNet       │ train_ekf.py │ 预测陀螺零偏     │ biasnet_weights.weights.h5     │
  ├───────────────┼──────────────┼──────────────────┼────────────────────────────────┤
  │ CovAdapterNet │ train_e2e.py │ 动态调整噪声大小 │ cov_adapter_weights.weights.h5 │
  └───────────────┴──────────────┴──────────────────┴────────────────────────────────┘

  validate_ekf.py 会把两个都加载到EKF里一起用：

  BiasNet → 去掉陀螺零偏 → 航向更准
  CovAdapterNet → 动态调噪声 → 转弯放松NHC，直道收紧
                                      ↓
                               6状态EKF推理

  所以流程是：
  train_ekf.py  →  BiasNet权重
                                ↘
                          validate_ekf.py  ← 看最终轨迹效果
                                ↗
  train_e2e.py  →  CovAdapterNet权重

  现在跑 python validate_ekf.py 就能看新训练的CovAdapterNet有没有改善效果了。

● EKF状态向量里有一个 bg（第6维），它是EKF内部维护的残余零偏估计：

  实际零偏 ≈ BiasNet预测 + EKF残差(bg)

  预测步:
    omega = gyro_z - BiasNet预测 - bg
    yaw += omega * dt

  BiasNet预测的是"主要零偏"，bg 是EKF在线微调的"残差补偿"。有GPS时，EKF会调整 bg
  让定位更准；丢GPS时，bg 就冻结了。

  所以图里三条线的关系：

- 蓝线（BiasNet预测）= 模型从IMU算出来的零偏
- 红线（EKF残差）= EKF在线微调的部分，通常接近0
- 绿线（合计）= 实际用于校正陀螺的总零偏
