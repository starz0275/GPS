## 一、基础术语

```plaintext
          ↑ Z (竖直向上，穿出纸面)
          |
          |
← Y 左————●———— Y 右
          |
          |
          ↓ X (车头向前)
```


| 名词                          | 解释                                                                                                                                                                                                                                       |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **roll / pitch / yaw**  | 姿态角（欧拉角）。``•  **roll（横滚角）** ：绕前进方向（X轴）的旋转，比如车辆左右倾斜。``•  **pitch（俯仰角）** ：绕横向轴（Y轴）的旋转，比如车辆上坡下坡。``•  **yaw（偏航角）** ：绕垂直轴（Z轴）的旋转，也就是车头朝向。 |
| **IMU（惯性测量单元）** | 一种传感器，通常包含加速度计和陀螺仪。可以测出三个方向的加速度和三个方向的角速度。                                                                                                                                                         |
| **6轴 IMU**             | 指能输出**3轴加速度 + 3轴角速度** 的 IMU。数据里就是 AccX/Y/Z 和 GyroX/Y/Z。                                                                                                                                                         |
| **加速度（AccX/Y/Z）**  | 测量 IMU 在三个方向上受到的**比力** （包括重力和车辆自身的加速度）。单位通常是 m/s² 或 g。                                                                                                                                          |
| **角速度（GyroX/Y/Z）** | 测量 IMU 绕三个轴旋转的**快慢** ，单位 rad/s 或 °/s。                                                                                                                                                                               |
| **GPS**                 | 全球定位系统，可以提供位置、速度、航向（`gps_theta`）等信息，但频率低（10 Hz 左右）且可能受遮挡。                                                                                                                                        |
| **gps_theta**           | GPS 给出的**航向角** （行进方向，通常相对于真北或 ENU 坐标系的 X 轴）。在你的方案里用它来监督 yaw。                                                                                                                                  |
| **gps_valid**           | 一个标志位，表示当前时刻 GPS 数据是否有效（比如没有信号或精度太差时为 False）。                                                                                                                                                            |
| **CMCC 六轴 bias**      | CMCC 应该是你手里的一个数据集或设备型号。“bias”指 IMU 的零偏（静止时输出的非零值）。这组数据主要用于学习/补偿 IMU 的固定误差。                                                                                                           |
| **车辆速度 v_ms**       | 车速（米/秒）。可以辅助判断车辆是否在运动，也可以帮助区分重力加速度和运动加速度。                                                                                                                                                          |

## 二、姿态与坐标系相关

| 名词                         | 解释                                                                                                                                                             |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **四元数**             | 一种用四个数 `[qw, qx, qy, qz]` 表示旋转的方法，没有奇异性（不像欧拉角在 pitch=±90° 时出问题）。网络输出四元数很方便。                                       |
| **欧拉角（Euler 角）** | roll/pitch/yaw 的统称，直观但要注意旋转顺序和奇异性。                                                                                                            |
| **旋转矩阵**           | 一个 3x3 的正交矩阵，左乘一个向量可以把向量从一个坐标系旋转到另一个坐标系。                                                                                      |
| **轴角（axis-angle）** | 用“一个方向 + 绕该方向转多少度”来表示旋转，通常不直接回归。                                                                                                    |
| **导航系 / 机体系**    | •**导航系（n-frame）** ：固定的参考坐标系，例如“东北天”（ENU）。``•  **机体系（b-frame）** ：固定在载体（车/IMU）上的坐标系，随载体运动。 |
| **ENU**                | 一种导航坐标系：**东（East）- 北（North）- 天（Up）** 。常用于地面车辆。                                                                                   |
| **重力对齐**           | 利用加速度计测量重力方向这一原理，约束姿态的 roll 和 pitch。静止时，加速度计测量到的方向就是重力方向；运动时需要剔除车体加速度。                                 |

## 三、模型与网络结构

| 名词                            | 解释                                                                                                         |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **窗口（window）**        | 一段连续的时间序列，比如 1~2 秒内的所有 IMU 数据。模型一次输入一个窗口的数据，输出这一个窗口中间时刻的姿态。 |
| **TCN（时间卷积网络）**   | 一种专门处理序列数据的卷积网络，能够看到过去和未来一个范围的信息，比 RNN 更稳定。                            |
| **1D CNN**                | 一维卷积神经网络，对时间序列的每一维做滑动卷积，提取局部特征。                                               |
| **GAP（全局平均池化）**   | 对整个时间维度取平均值，把每个特征通道变成一个数，常用于网络最后一层，减少参数量。                           |
| **BiasNet / DeepBiasNet** | 你仓库里已有的模型结构，用于学习 IMU 偏置或姿态，方案建议在上面修改。                                        |

## 四、损失函数与约束（训练用的“惩罚项”）

| 名词                            | 解释                                                                                                                                           |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **GPS yaw 损失**          | 让模型预测的 yaw 角尽量接近 GPS 给出的航向角。因为角度有周期性（-180° 和 180° 是同一个方向），所以要用 `sin²(差值)` 或“wrap 后算 MSE”。 |
| **sin² 差值**            | 一种角度差损失。`sin²(Δθ)` 在 Δθ=180° 时和 0° 时的值一样（因为周期 π），避免了 179° 和 -179° 相差 358° 导致的大误差问题。         |
| **wrap_angle**            | 把角度限制到 `[-π, π)` 或 `[0, 2π)` 范围内，消除 360° 跳变。                                                                           |
| **重力对齐损失**          | 模型预测出姿态后，把导航系的重力矢量（一般是 [0,0,g]）旋转到机体系，应该和加速度计测量的方向一致。用于约束 roll/pitch。                        |
| **积分一致性损失**        | 连续两帧的姿态变化应该等于陀螺仪角速度在这一段时间内的积分。强制这个一致，可以减少抖动。                                                       |
| **四元数/矩阵正规化损失** | 四元数必须是单位长度，旋转矩阵必须是正交矩阵。加一个损失让它们接近理想的数学形式。                                                             |
| **平滑损失**              | 相邻帧的姿态变化应该平缓，防止突然跳变。                                                                                                       |
| **MSE（均方误差）**       | 一种常用的回归损失：`mean( (预测-真值)² )`。                                                                                                |

## 五、训练与评估

| 名词                         | 解释                                                                                                                                  |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **归一化**             | 将输入特征（比如加速度值）减去均值、除以标准差，使其数值范围相似，便于训练。你的仓库里有 `normalization_stats_data0109.json` 文件。 |
| **验证段**             | 训练时不用来更新模型，只用来检查模型性能的数据部分，防止过拟合。你这里验证段是 Data05（4 圈跑道）。                                   |
| **RMSE（均方根误差）** | 评估指标：`sqrt( mean((预测-真值)²) )`，对误差敏感。                                                                               |
| **中位数误差**         | 评估指标：把所有误差排序，取中间那个值，更能抵抗异常值。                                                                              |
| **代理指标**           | 因为没有真值，只能用其他可计算的量间接评估。比如 roll/pitch 没有真值，就看重力对齐误差。                                              |

## 六、其他相关术语

| 名词                          | 解释                                                                                                                              |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **Gram-Schmidt 正交化** | 一种数学方法，可以将一个不完美的旋转矩阵“拉”回到正交矩阵（逼近真实旋转矩阵）。                                                  |
| **SVD（奇异值分解）**   | 也可以用来对矩阵做正交化修正。                                                                                                    |
| **低动态 / 高速**       | •**低动态** ：车辆几乎匀速直线运动或静止，加速度计主要测重力。``•  **高速** ：速度较大时，GPS 航向通常更准。 |
