# 姿态角预测方案

## 一、目标

- 从 Data0109 的 IMU 数据学习完整姿态。
- 最终输出 `roll/pitch/yaw`。
- 由于 Data0109 不提供直接的姿态真值，训练需要弱监督和物理一致性约束。

## 二、可用信息

- IMU 6 轴：`AccX_g`, `AccY_g`, `AccZ_g`, `GyroX_degs`, `GyroY_degs`, `GyroZ_degs`
- 轮速：`VehicleSpeed_ms`
- GPS 航向：`gps_theta`（从 GNSS 位移计算）
- `gps_valid`：GPS 航向有效掩码
- CMCC 六轴 bias：当前主要用于偏置学习，非直接姿态标签

## 三、方案核心

1. 训练时采用 `四元数` 作为姿态表示。
2. 训练目标包括：
   - GPS 航向监督 yaw
   - 重力对齐监督 roll/pitch
   - IMU 连续性与归一化约束
3. 推理时将四元数转换为 `roll/pitch/yaw`。

## 四、数据准备

- 使用 `data0109_loader.py` 加载 Data0109 段。
- 训练段：
  - `Data01_0109_1圈8字+2圈跑道`
  - `Data02_0109_2圈8字+2圈跑道`
  - `Data03_0109_1圈跑道+2圈8字`
  - `Data04_0109_2圈跑道+2圈8字`
- 验证段：`Data05_0109_4圈跑道`
- 采用 10 Hz 对齐窗口，建议窗口大小 `100-200` 帧。
- 输入特征：`(AccX/Y/Z, GyroX/Y/Z, VehicleSpeed_ms)`。

## 五、弱监督构造

### 5.1 GPS yaw 监督

- 仅在 `gps_valid` 为真时使用。
- 使预测 yaw 与 GPS 航向保持一致。

### 5.2 重力对齐监督

- 在低动态车辆段或速度较低时，
- 将预测姿态转换出的机体重力方向与测得加速度方向对齐。

### 5.3 归一化与正交性

- 四元数输出需归一化。
- 避免直接回归旋转矩阵。

## 六、模型设计

- 输入：滑动窗口 `X ∈ ℝ^{W×7}`。
- 输出：四元数 `q ∈ ℝ^4`。
- 网络结构：基于 1D 卷积 + 全局平均池化 + 全连接。
- 训练时只保留有 GPS 或重力监督的窗口。

## 七、损失设计

- yaw loss：`1 - cos(pred_yaw - gps_theta)`，对 GPS 有效窗口加权。
- gravity loss：机体重力方向与加速度归一化方向的夹角损失。
- 总损失：`yaw_loss + α * gravity_loss`。

## 八、训练与评估

- 训练时先以 GPS yaw 和重力对齐为主。
- 验证时关注：
  - yaw RMSE 与 GPS 航向误差
  - roll/pitch 重力一致性指标
- 最终推理：从四元数转出 `roll/pitch/yaw`。

## 九、实现文件

- 新脚本：`scripts/train_attitude_from_imu.py`
- 文档：`docs/姿态角预测方案.md`

## 十、执行方案

1. 加载 Data0109 训练/验证数据
2. 读取 `preprocessed_data/normalization_stats_data0109.json`
3. 构造弱监督标签：GPS yaw + 重力对齐
4. 训练四元数姿态网络
5. 保存权重，并输出验证结果

## 结果


当前模型效果（验证集 Data05_4圈跑道，1693 个窗口）：



- Roll/Pitch 效果很好：误差在 1° 左右，对于车辆姿态估计来说是可用的精度。车辆正常行驶时 roll 范围约
  ±4°、pitch 约 ±1.5°，模型的误差相对于信号范围是合理的。
- Yaw 中等偏可用：5° 的航向误差在 GNSS+IMU 融合系统中偏大（好的融合系统能做到
  1-2°），但考虑到模型的输入 GPS
  航向本身就带有几度的噪声（由位移差分计算，低速时更差），这个结果是合理的。
- yaw 误差大头来自输 入：模型依赖 GPS 航向作为绝对参考，如果 GPS 航向本身噪声大，模型输出也不会更好。

  对比项目里已有的方案

  你的 EKF（validate_ekf.py）用 gyro Z 积分 + BiasNet 零偏修正 + GPS 更新，yaw 误差一般 <
  2°。AttitudeNet 的优势是同时输出 roll 和 pitch（EKF 里没有这两个量），牺牲了一些 yaw 精度。
