## 论文核心：AI-IMU Dead Reckoning

### 整体框架

```
IMU原始数据 → LSTM网络估计零偏 → 去零偏后送入EKF → 航位推算结果
                ↑                        ↑
          物理模型约束              GNSS真值训练
```

论文的关键创新是 **用深度学习替代 EKF 中的状态转移模型** ，同时学习 IMU 零偏。

### 1. 零偏估计（Section 3.3）

LSTM 网络接收 **归一化后的 IMU 序列** ，输出 **零偏估计** ：

```
输入: Xn = [xn1, xn2, ..., xnT]  (归一化后的历史 IMU 测量)
输出: bn ∈ ℝ⁶  (陀螺仪3轴 + 加速度计3轴零偏)
```

零偏处理方式（Eq. 22-23）：

```
ω_corr = ω_meas − bg      (陀螺仪去零偏)
f_corr = f_meas − ba      (加速度计去零偏)
```

 **训练策略** （Algorithm 1）：

* 不是直接用零偏真值做监督
* 而是将零偏估计送入 EKF，EKF 输出轨迹与 GNSS 轨迹对比
* **损失 = EKF预测轨迹 vs GNSS真实轨迹** （间接监督零偏）
* 这样网络学到的零偏是为了让 EKF 轨迹最准，而非拟合 CMCC 零偏标签

### 2. 安装角（Section 2.1 & 3.3）

论文定义了三个坐标系：

| 坐标系              | 符号 | 说明                  |
| ------------------- | ---- | --------------------- |
| 导航系 (Navigation) | n    | 局部切面坐标系        |
| 车体系 (Body)       | b    | 固连于车辆，x前y右z下 |
| IMU 系 (IMU)        | —   | 固连于 IMU 传感器     |

**安装角**就是 IMU 系到车体系的固定旋转 `R_imu→body`。论文里假设安装误差很小（Eq. 16 附近提到 "installation errors are small"），即 IMU 轴基本对齐车体轴。

对于你的场景，安装角是一个 **固定旋转矩阵** （3个角度参数），表示：

```
v_body = R_install × v_imu
```

### 3. EKF 状态量（Table 1）

| 状态        | 说明         |
| ----------- | ------------ |
| 航向角      | EKF 直接估计 |
| 位置 (x, y) | 航位推算积分 |
| 陀螺仪零偏  | 由 LSTM 学习 |
| 加速度零偏  | 由 LSTM 学习 |
| 航向角误差  | EKF 自身估计 |

## 对你项目的启发

### 零偏方面

* 你的项目已有 `BiasNet`（从 Data0109 CMCC 零偏学 6 轴零偏），思路和论文一致
* 论文的亮点是 **端到端训练** ：不直接监督零偏，而是让零偏服务于轨迹精度
* 你的 `train_ekf.py` 已经在做类似的事（EKF 轨迹损失 + 零偏修正）

### 安装角方面

* 安装角是 **3 个固定参数** （IMU 相对于车体的安装旋转）
* 可以作为**可训练参数**加入网络，或通过标定实验事先测好
* 如果安装角已知，只需做一次旋转变换；如果未知，可以当作网络的可学习参数一起优化

### 你当前的 AttitudeNet

* 你的 TCN 姿态网络直接从 IMU 回归四元数，思路更偏端到端
* 论文的方法更偏 "EKF + NN 辅助"，两者可以互补
* 如果要参考论文的思路，可以考虑：**AttitudeNet 输出的安装角为可学习固定参数，零偏由网络学习，最终融合进 EKF**
