# 全局 3 安装角 + 固定零偏微调的轨迹优化方案

## Summary

- 安装角按你的目标改为 **3 个全局可训练固定参数**：`install_roll/pitch/yaw`，所有窗口共享。
- 零偏采用 **6 个全局固定零偏 + 小残差网络微调**：最终输出以全局 6 零偏为标定结果，残差只用于补偿慢漂。
- 训练目标是：不断调整 **6 零偏 + 3 安装角 + 小残差网络**，让积分出来的轨迹和 GNSS 真值轨迹误差变小。

## Key Changes

- 先修 GNSS/ENU 真值：

  - 用 `GpsFlag > 0`、经纬度非 0、经纬度有限、速度跳变阈值清洗 GNSS。
  - ENU 参考点取第一个有效 GNSS 点。
  - 无效 GNSS 点不参与 loss，只对有效点插值生成连续轨迹。
  - 修复后 Data01/Data02/Data05 的 ENU 范围必须回到百米级。
- 重构训练模型：

  - 全局可训练参数：
    - `bias_base[6] = ba_x, ba_y, ba_z, bg_x, bg_y, bg_z`
    - `install_rpy[3] = roll, pitch, yaw`
  - 网络只输出小零偏残差 `bias_residual[6]`。
  - 实际使用零偏：
    - `bias_used = bias_base + residual_scale * bias_residual`
  - `install_rpy` 不由窗口网络输出，而是全局共享，并且直接参与每个窗口的 IMU 坐标旋转和轨迹积分。
- 轨迹推算链路：

  - 原始 IMU 还原物理单位。
  - 减去 `bias_used`。
  - 用 `R_body_imu(install_roll, install_pitch, install_yaw)` 把 IMU 系旋到车体系。
  - 用 `gyro_body_z` 积分航向。
  - 用车速和航向积分平面轨迹。
  - 用预测轨迹和 GNSS ENU 真值计算位置 loss。
  - 航向可加辅助 loss，但主目标是轨迹位置误差。
- 输出结果：

  - 保存 `bias_angle_v3_result.json`：
    - 6 个全局零偏
    - 3 个全局安装角
    - residual 均值/标准差/最大值
    - train/val 轨迹 mean/RMSE/P95
    - heading mean/RMSE/P95
  - 生成 Data01-Data05 轨迹图：
    - GNSS 真值
    - 只用全局固定参数的轨迹
    - 全局参数 + 残差微调轨迹

## Test Plan

- 数据验证：
  - 加载 Data01-Data05，确认 ENU span 不再出现千万米。
  - 检查有效 GNSS 点比例和轨迹图是否正常。
- 训练验证：
  - v3 训练后验证集轨迹 RMSE 应低于当前 v2 的约 `19.83 m`。
  - 对比 fixed-only 和 fixed+residual，确认 residual 是改善轨迹而不是乱拟合。
- 参数验证：
  - 安装角不使用 CMCC 监督，只用轨迹误差优化。
  - 训练后用 CMCC `rbv_pitch/rbv_yaw` 做评估对比；roll 没有 CMCC 真值，只报告学习结果。
  - 零偏输出以 `bias_base` 为最终固定标定值。

## Assumptions

- “预测安装角”定义为：通过训练学习 3 个全局固定安装角，并在轨迹积分中持续参与优化。
- 安装角不按窗口变化，因为物理安装角应是固定值。
- 零偏允许小残差微调，因为真实 IMU 零偏可能随时间慢漂。
- 安装角默认限制在小角度范围，例如 `[-10°, 10°]`，防止不可观方向发散。
