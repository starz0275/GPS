# Data0109 CMCC 零偏训练说明

本文说明如何用 `CMCC_result.txt` 真值监督微调 BiasNet，使六轴零偏 `[ba_x, ba_y, ba_z, bg_x, bg_y, bg_z]` 接近 CMCC 稳态输出。

相关脚本：


| 脚本                       | 作用                                                         |
| ------------------------ | ---------------------------------------------------------- |
| `data0109_loader.py`     | 加载 IMU/GNSS/轮速 + CMCC 六轴标签，生成 `cmcc_ok` / `cmcc_stable` 掩码 |
| `train_biasnet_cmcc.py`  | 三阶段 CMCC 微调（默认 DeepBiasNet）                                |
| `evaluate_bias_cmcc.py`  | 滑窗推理、对比图、MAE 与 JSON 日志                                     |
| `biasnet_postprocess.py` | 推理后处理：仅 stable 段输出 + 平滑限幅                                  |
| `ekf_navigator.py`       | `BiasNet` / `DeepBiasNet` 定义与 `make_biasnet()` 工厂          |


更短的工程入口见 `docs/CLAUDE.md`。

---

## 1. 数据与划分

数据根目录：`Data0109数据及零偏/<段名>/`，每段含 `*_IMU.txt`、`*_GNSS.txt`、`*_VehicleSpeed.txt`、`*_CMCC_result.txt`。


| 用途  | 段名                                 |
| --- | ---------------------------------- |
| 训练  | Data01～Data04（四段）                  |
| 验证  | **Data05_0109_4圈跑道**（不参与梯度，仅早停与评估） |


对齐：10 Hz（`TARGET_DT = 0.1 s`）。CMCC 六列与网络输出一一对应：`acc_bias_x/y/z` [g]，`gyro_bias_x/y/z` [deg/s]。

归一化统计：`preprocessed_data/normalization_stats_data0109.json`（由训练段 IMU 七通道 mean/std 计算）。

---

## 2. 何时参与训练（掩码）

零偏在 CMCC 标定收敛前会爬升，起步静止段也不应学习。因此不用「整段文件」，而用两级掩码：

```
cmcc_ok      = calibration_type ≥ 3 且纬度有效
cmcc_stable  = cmcc_ok 之后，每段连续区间内再等待 settle_s（默认 60 s）
```

示意（Data05）：

```
0 ───── 静止/未标定 ─────► ~106 s  cmcc_ok 开始
~106 s ─── CMCC 收敛期（60s）───► ~166 s  cmcc_stable 开始（绿底）
~166 s ───────────────────────► 段末   仅此段参与训练与主评估
```

实现：`data0109_loader.cmcc_settle_mask()` / `cmcc_stable_mask()`。  
阶段 2/3 要求滑窗 **整窗** 都在 `cmcc_stable` 内（`require_full_window=True`），即窗口右端帧在 stable 且前 `window_size-1` 帧也在 stable。

可调参数：

- 常量 `CMCC_SETTLE_S = 60.0`（`data0109_loader.py`）
- 命令行 `--settle-s 50`（训练与评估一致）

## 3. 模型结构

通过 `make_biasnet(arch, window_size)` 创建。

### 3.1 Shallow — 原 `BiasNet`（`--arch shallow`）

- 窗口：**30 帧（3 s）**
- 结构：3 层因果 Conv1D + GlobalAveragePooling + Dense(6)
- 参数量约 **5k**
- 可从 `trained_models/biasnet_weights.weights.h5`（`train_ekf.py` GPS 伪标签）初始化

### 3.2 Deep — `DeepBiasNet`（**默认** `--arch deep`）

- 窗口：**200 帧（20 s）**
- 结构：Stem Conv1D → 6 个 TCN 残差块（dilation 1/2/4/8/16/32）→ GAP → Dense(64) → Dense(6)
- 感受野约 **253 帧**，覆盖整窗
- 参数量约 **91k**
- **从零训练**（与浅层权重结构不兼容，不加载旧 30 帧权重）

输入：`(batch, window, 7)`，七通道为归一化后的 `AccX/Y/Z_g, GyroX/Y/Z_degs, VehicleSpeed_ms`。

输出：6 维线性 `raw`，推理时再经 `tanh` 限幅：

- acc：`±0.25 g × tanh(raw[0:3])`
- gyro：`±2.0 °/s × tanh(raw[3:6])`

## 4. 三阶段训练流程

入口：`python train_biasnet_cmcc.py`（默认 `arch=deep`, `window_size=200`）。


| 阶段  | 掩码                     | 标签                                              | 损失                            | 学习率        | 输出权重                                 |
| --- | ---------------------- | ----------------------------------------------- | ----------------------------- | ---------- | ------------------------------------ |
| 1   | `cmcc_stable`          | CMCC 瞬时六轴                                       | 六轴等权 Huber                    | deep: 5e-4 | `biasnet_weights_cmcc_s1.weights.h5` |
| 2   | `cmcc_stable` + 标签中值平滑 | 同上                                              | 六轴等权 Huber                    | deep: 1e-4 | `biasnet_weights_cmcc.weights.h5`    |
| 3   | `cmcc_stable`          | **acc**：段内 stable **末 60 s 均值**；gyro：平滑后瞬时 CMCC | tanh 对齐的 acc MSE + gyro Huber | 2e-5       | 覆盖最终权重 + 写 acc 校准 JSON               |


阶段 3 要点：

- `acc_constant_tail=True`：每段 stable 末 60 s 的 `ba_x/y/z` 均值作为常值标签（贴近「稳态常偏」）。
- `acc_sample_dup=2`：acc 样本重复，加强 acc 梯度。
- 损失在 **物理量（含 tanh）** 上计算，与评估推理一致。
- 结束后 `fit_acc_calib()`：在训练段 stable 上估计 acc 系统残差 → `biasnet_cmcc_acc_calib.json`。

早停：监控 `val_loss`，`ModelCheckpoint` 保存最优权重；`ReduceLROnPlateau` 衰减学习率。

训练元数据写入：`trained_models/biasnet_info_cmcc.json`（含 `arch`, `window_size`, 各阶段 `best_val_mae`）。

## 5. 推理与后处理（评估默认开启）

`evaluate_bias_cmcc.predict_bias_6d()` 流程：

1. 滑窗 DeepBiasNet → `tanh` 限幅
2. acc 加上 `biasnet_cmcc_acc_calib.json` 偏移
3. **仅 `cmcc_stable` 段**：EMA（约 2 s）+ 步长限幅（不含中值滤波）
4. **非 stable 段**：预测置 `NaN`（图上不画蓝线）

因此：

- **训练**：只在绿底（stable）内算 loss  
- **评估 MAE**：默认只在 stable 内统计（段外 NaN 不参与）  
- **EKF 实车**（`ekf_navigator._predict_bias`）：目前仍为浅层逻辑，未自动套用上述掩码/平滑；若需一致需另行接入 `biasnet_postprocess`

## 6. 常用命令

```bash
# 全流程三阶段（默认 deep + window=200 + settle=60s）
python train_biasnet_cmcc.py

# 仅重跑阶段 3（需已有 biasnet_weights_cmcc.weights.h5）
python train_biasnet_cmcc.py --acc-only

# 使用旧浅层网络（3 s 窗，需有 train_ekf 基线权重）
python train_biasnet_cmcc.py --arch shallow --window-size 30

# 稳定等待改为 50 s
python train_biasnet_cmcc.py --settle-s 50

# 评估（自动从 biasnet_info_cmcc.json 读 arch / window_size）
python evaluate_bias_cmcc.py
python evaluate_bias_cmcc.py --segment "Data05_0109_4圈跑道"

# 关闭推理平滑或段外掩码（调试用）
python evaluate_bias_cmcc.py --no-smooth
python evaluate_bias_cmcc.py --no-infer-mask
```

单跑某一阶段：`python train_biasnet_cmcc.py --stage 1`（或 `2`）。

---

## 8. 指标怎么读

- **MAE**：stable 段内逐帧绝对误差均值，越小越好。  
- **末 60 s 均值差**：`pred_mean - cmcc_mean`，看稳态是否系统性偏高/偏低。  
- **val_mae**（训练日志）：验证集 Data05 上的 Keras `mae`；阶段 3 因 loss 定义不同，数值不宜与阶段 1/2 直接比大小。

## 10. 常见问题

**Q：为什么图上 stable 之前没有蓝线？**  
A：评估默认 `mask_outside=True`，段外预测为 NaN，避免静止/收敛期误导。

**Q：重新训练会覆盖哪些文件？**  
A：`biasnet_weights_cmcc*.weights.h5`、`biasnet_info_cmcc.json`、`biasnet_cmcc_acc_calib.json` 以及 `cmcc_bias_compare_*.png`、新的 `training_logs/cmcc_eval_*.json`。

**Q：阶段 1 能否用整段 `cmcc_ok`？**  
A：`--stage1-cmcc-ok` 可以，但会包含 CMCC 爬升段，一般不推荐。

**Q：EKF 怎么用新模型？**  
A：`EKFNavigatorNP` 构造时传入与 `biasnet_info_cmcc.json` 一致的 `window_size`，并加载 `biasnet_weights_cmcc.weights.h5`；需将 `BiasNet(...)` 改为 `make_biasnet('deep', 200)` 才与权重匹配（当前 navigator 仍写死 `BiasNet`，接入深模型时要改一行实例化）。
