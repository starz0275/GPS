# 零偏和安装角学习实现说明

## 一、目标

当前这套实现的目标是：

- 从 `Data0109` 的 IMU + 轮速 + GNSS 轨迹中学习
  - `6` 个全局固定零偏
  - `3` 个全局固定安装角 `roll/pitch/yaw`
- 同时允许一个很小的零偏残差随时间微调
- 让用这些参数积分出来的轨迹尽量贴近 GNSS 真值轨迹

这套方法不再直接用 CMCC 安装角做训练监督，而是主要通过轨迹误差反向优化。CMCC 的 `rbv_pitch/rbv_yaw` 只作为训练后的参考评估。


## 二、整体思路

训练链路如下：

```text
IMU窗口(200帧) + 轮速
    -> TCN网络
    -> 输出小零偏残差 residual_bias(6)

全局固定参数:
    bias_base(6)
    install_rpy(3)

实际使用参数:
    bias_used = bias_base + residual_bias
    install = install_rpy

物理链路:
    原始IMU -> 去零偏 -> 按安装角旋到车体系
           -> 用 gyro_z 积分航向
           -> 用车速 + 加速度积分轨迹
           -> 与 GNSS ENU 真值对比
           -> 反向优化
```

核心思想是：

- `bias_base` 和 `install_rpy` 是最终想要的“标定量”
- `residual_bias` 只负责补偿慢漂，不应该喧宾夺主
- 损失函数的主目标是轨迹位置误差，航向误差是辅助项


## 三、数据怎么准备

### 1. 数据源

`data0109_loader.py` 负责加载以下 5 段数据：

- 训练集：
  - `Data01_0109_1圈8字+2圈跑道`
  - `Data02_0109_2圈8字+2圈跑道`
  - `Data03_0109_1圈跑道+2圈8字`
  - `Data04_0109_2圈跑道+2圈8字`
- 验证集：
  - `Data05_0109_4圈跑道`

### 2. GNSS 真值修复

这次实现里先修了轨迹真值问题。

旧版本的问题是：

- GNSS 文件前面有不少 `Latitude=0, Longitude=0`
- 这些点直接参与插值后，会把 ENU 拉到千万米量级

现在 `data0109_loader.py` 的处理方式是：

- 用 `GpsFlag > 0`
- 用 `Latitude_deg > 1`
- 用 `Longitude_deg > 1`
- 用有限值判断
- 再加速度跳变门限去掉异常 GNSS 点

然后：

- 取第一个有效 GNSS 点作为 ENU 参考点
- 对有效 GNSS 做插值，得到连续 `enu_x/enu_y`
- 保留两个掩码：
  - `gps_pos_valid`：位置真值有效
  - `gps_valid`：航向真值有效

修完之后，五段数据的 ENU 范围都回到了大约 `195m x 160~173m`，不再出现千万米异常。


## 四、模型怎么实现

实现文件是 [scripts/train_bias_angle.py](/C:/Users/nxj/Desktop/NewGPS/GPS/scripts/train_bias_angle.py)。

### 1. 模型结构

模型 `BiasAngleNet` 分成两部分：

- 全局可训练参数
  - `bias_base[6]`
  - `install_raw[3]`
- 窗口网络
  - `Conv1D + TCN blocks + GAP + Dense`
  - 输出 `residual_bias[6]`

其中：

- `bias_base` 直接表示固定零偏
- `install_raw` 经过 `tanh` 和角度范围限制后，得到全局固定安装角 `install_rpy`
- `residual_bias` 经过缩放后，范围很小：
  - 加速度残差默认上限约 `0.02 g`
  - 陀螺仪残差默认上限约 `0.02 deg/s`

### 2. 安装角处理

安装角使用 `roll/pitch/yaw` 三个角，作为一个固定的 `IMU -> Body` 旋转。

代码里用的是 `ZYX` 欧拉角形式的旋转矩阵，把：

- 加速度 `acc_imu`
- 陀螺仪 `gyro_imu`

从 IMU 坐标系旋转到车体系。

### 3. 轨迹积分

积分逻辑也在 [scripts/train_bias_angle.py](/C:/Users/nxj/Desktop/NewGPS/GPS/scripts/train_bias_angle.py)：

- 先还原 IMU 的物理单位
- 减去 `bias_used`
- 按安装角旋转到车体系
- 用 `gyro_body_z` 积分航向
- 用 `v_ms` 作为主速度
- 用车体系加速度投影到导航系做短时修正
- 得到 `east/north/heading`

### 4. 损失函数

训练的总损失主要由三部分组成：

```text
total_loss =
    position_loss
  + head_weight * heading_loss
  + residual_l2_weight * residual_l2
  + bias/install 的小先验正则
```

解释如下：

- `position_loss`
  - 预测轨迹和 GNSS 轨迹的误差
  - 只在 `gps_pos_valid=True` 的点上计算
- `heading_loss`
  - 预测航向和 GNSS 航向的误差
  - 只在 `gps_valid=True` 的点上计算
- `residual_l2`
  - 压制残差不要过大
- `bias/install` 正则
  - 防止固定零偏和安装角发散
  - 安装角还加了一个“小角度先验”，防止只靠轨迹误差时跑到边界


## 五、评估怎么做

评估文件是 [scripts/evaluate_bias_angle.py](/C:/Users/nxj/Desktop/NewGPS/GPS/scripts/evaluate_bias_angle.py)。

这里有两个重点：

### 1. 评估方式和训练一致

训练时不是整段一把积分，而是：

- 每个窗口从窗口起点 GNSS 初值开始积分
- 窗口长度 `200` 帧

所以评估时也改成了同样的滑窗方式：

- 先对每个窗口预测
- 再把多个窗口在同一时刻的结果聚合平均

这样评估指标才能和训练目标一致。

### 2. 评估输出两种轨迹

评估图里会同时画：

- `Fixed-only`
  - 只用全局 `bias_base + install_rpy`
- `Fixed + residual`
  - 在全局固定参数基础上，再加上网络预测的小残差

这样就能看出来：

- 固定标定量本身能做到什么程度
- 小残差到底有没有帮助

## 七、当前结果

当前正式训练结果保存在 `bias_angle_v3_result.json` 中，主要数值如下：

### 1. 学到的固定参数

```text
安装角 [deg]
roll  = -1.1527
pitch = -0.2681
yaw   =  0.4187

加速度零偏 [g]
ba_x =  0.4751
ba_y =  0.2810
ba_z = -0.1960

陀螺仪零偏 [deg/s]
bg_x = -0.0390
bg_y =  0.0496
bg_z = -0.1049
```

### 2. 轨迹效果

验证集窗口级统计：

```text
val fixed-only:
position rmse = 11.59 m

val with residual:
position rmse = 11.28 m
```

整段滑窗聚合后的 Data05 图上：

```text
fixed-only rmse = 6.74 m
fixed + residual rmse = 6.60 m
```

说明：

- 固定零偏 + 固定安装角已经能把轨迹拉到比较接近
- 小残差可以继续提供一点改善
- 但改善幅度不算特别大，说明主效果还是来自固定参数

### 3. 和 CMCC 安装角的关系

当前没有用 CMCC 安装角做训练监督，只在训练后对比：

```text
CMCC pitch mean = -1.6791 deg
CMCC yaw mean   = -1.6873 deg

当前模型:
pitch = -0.2681 deg
yaw   =  0.4187 deg
```

也就是说：

- 轨迹拟合已经比较好
- 但学出来的安装角不一定等于 CMCC 的安装角
- 这说明轨迹误差里，安装角和零偏之间仍然存在一定的互相补偿关系


## 八、当前结论

现在这套实现已经完成了下面几件事：

- GNSS 真值修复完成
- 全局 6 零偏 + 全局 3 安装角训练链路打通
- fixed-only 与 residual 两套轨迹都能评估
- 结果文件和图都能自动生成

当前更适合把这套模型理解为：

- 一个以“轨迹最优”为目标的参数学习器
- 而不是一个严格等价于 CMCC 物理标定值的求解器

如果后面希望“轨迹准”和“安装角/零偏更接近物理真值”同时成立，下一步通常会考虑：

- 加更强的安装角先验
- 引入静止段/低动态段约束
- 对零偏和安装角增加更明确的物理可观性约束
