## 把现在的预测起伏大归因到三件事：

窗口太短：3s（30 帧）只能看到瞬时 IMU 抖动，无法判断"基线"在哪  
每帧独立预测：没有任何时间一致性约束，相邻帧可以差得很多  
网络太浅：3 层 Conv + 全局平均池化，~5k 参数，容量小到记不住"段内常值"模式  
标签是瞬时值：除阶段3 外，gyro 仍跟 CMCC 瞬时输出，把 CMCC 自身的小抖动也学进来了

## 调研结果（业界做法）


| 论文                                     | 思路                                                      | 对我们的启发                          |
| -------------------------------------- | ------------------------------------------------------- | ------------------------------- |
| **IPNet** (2025, arxiv 2503.12527)     | 残差 CNN + 序列建模 + **每窗口预测 n 个值取均值**；用**整段 IMU 平均零偏**作训练标签 | 直击"零偏=段内平均值"这一物理观察              |
| **VIO Deep Bias Factors** (2211.04517) | LSTM / Transformer 处理 **1s 窗口 + 100 个历史窗口**（共 100s 历史）  | gyro 慢变 → Transformer 在 z 轴误差最低 |
| **TLIO / RSS25 NDPs**                  | TCN 1–2s 窗口 + 序列到序列预测                                   | 与现状最接近，是渐进升级路径                  |
| **NODE** (2504.09495)                  | 显式建模 `db/dt = f(...)`，连续动力学                             | 数学上最贴近物理，但工程量大                  |
| **Diffusion** (2505.11763)             | 把 bias 建模成 IMU 条件概率分布                                   | 太复杂，先不考虑                        |


## 推荐方案（按性价比排序）

### 第一档 — 改动可控、预期收益大

**1. 扩窗 + 深 TCN（最该先做）**

- `WINDOW_SIZE: 30 → 200`（20s @ 10Hz），让网络看到 baseline
- 堆叠 dilation 1/2/4/8/16 的空洞卷积，感受野到几十秒
- 加 BN + 残差连接，参数 5k → ~50k
- 维持 causal、实车部署友好

**2. 时间一致性损失**
训练时给相邻窗口预测加平滑约束：

```
L = L_reg(ŷ_t, y_cmcc) + λ · ||ŷ_t − ŷ_{t−1}||²
```

等价于把"零偏慢变"作为先验注入梯度，比推理后 EMA 更彻底。

**3. 长窗手工特征**
当前 7 通道输入再拼 3 列：最近 60s 的 acc/gyro **滚动均值**和**静止占比**。让网络直接看见"baseline"而不是只看到瞬时波动。

**4. 序列平均标签（IPNet 思路扩展到 gyro）**
现在阶段3 acc 用 stable 末 60s 均值；可以**给 gyro 也加一组"段内 stable 平均"标签**，作为辅助监督头：

```
y = α · CMCC_瞬时 + (1−α) · CMCC_stable段均值
```

### 第二档 — 引入循环或注意力

**5. CNN + GRU 流式推理**

- 前端 TCN 提局部特征
- 后端单向 GRU 维持隐状态
- 推理时**跨窗口传递 GRU 状态**（30 分钟一直跑），慢变信息天然累积
- 参数 ~30k，推理仍然实时

**6. 双头解码（acc/gyro 分支）**
共享 encoder → 两个独立 head：

- acc head：常值偏置任务，权重大、平滑约束强
- gyro head：跟踪慢变，权重侧重 `bg_y`（当前最差）

避免六轴抢梯度。当前 `bg_y` MAE 0.05、`bg_z` 0.027，分头后两者可以独立调权重。

**7. 多尺度 Transformer 编码器**

- 短窗 token：每 2s 子窗口的 IMU 统计 → 30 个 token 覆盖 60s history
- 6 头自注意力 + Dense(6)
- 参数 100–300k，长依赖能力最强
- 验证集少时可能过拟合，需要更多数据增强

### 第三档 — 物理一致性

**8. NODE 显式 bias 动力学**

- 建模 `db/dt = f_θ(IMU, b)`，b 随时间积分演化
- 训练时积分整段并匹配 CMCC 轨迹
- 优点：天然平滑、可外推；缺点：工程量大，需要可微 ODE solver

**9. 输出不确定度**

- 输出 `(μ, log σ²)`，损失改 Gaussian NLL
- 推理时 `σ` 大就降权 / 更平滑
- 给 EKF 接入时还能作为动态 Q 矩阵权重

## 我的建议：先做 1 + 2 + 6

理由：

- **方案 1**（扩窗 + 深 TCN）是几乎所有更先进方法的基础，单独就能带来主要收益
- **方案 2**（时序平滑损失）能在训练阶段就压住起伏，比推理后处理更"治本"，且代码量小
- **方案 6**（双头解码）正好解决你之前提的 `bg_y` 和 `ba_z` 偏差不对称问题
- 这三个组合下来网络仍然是纯前馈 CNN，**部署、训练成本都可控**，参数 ~50k 量级
- 如果做完仍不够好，再上 GRU / Transformer
